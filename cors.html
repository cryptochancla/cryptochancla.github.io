“
En fait cryptochancla.herokuapp.com c'est juste un proxy CORS https://github.com/cryptochancla/cors-anywhere

Pour faire une requete signée sur l'API Binance, faut un header custom : "X-MBX-APIKEY" et comme le serveur de l'api binance n'a pas configuré de CORS pour fonctionner avec un requete fleshlight emise par le navigateur bah ça bloque.

Ca fait chier parce que oui c'est le seul truc qui peut faire douter d'un point de vue extérieur. Pour cette raison j'ai donc ajouté un input pour insérer son propre lien proxy CORS si vous ne voulez pas utiliser celui de cryptochancla.

Il faut créer son propre proxy CORS (très facile et rapide sur herokuapp) parce que les proxy CORS qu'on trouve sur google, leur IP a été trop utilisée sur binance, du coup "HTTP 403 return code is used when the WAF Limit (Web Application Firewall) has been violated."
”

Conclusion

Vous pouvez utiliser mon proxy CORS (https://cryptochancla.herokuapp.com/) qui est un simple fork de https://github.com/Rob--W/cors-anywhere
Dans ce cas rien à faire. Laissez le champ proxy CORS vide.

Ou alors, vous pouvez créer votre proxy CORS et saisir son url dans le champ proxy CORS.

Comment créer son proxy CORS en 2 minutes :

  • Inscrivez-vous/Connectez-vous sur herokuapp.com, choisissez NodeJS
  • Inscrivez-vous/Connectez-vous sur github.com, forkez le projet https://github.com/Rob--W/cors-anywhere
  • Synchronisez votre herokuapp avec votre dépot github que vous venez de créer/forker
  • Activez le déploiement automatique sur herokuapp
